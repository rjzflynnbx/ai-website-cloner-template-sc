# New Site Checklist — Adding a Site to Industry Verticals

Every file and config change needed to add a cloned site to an Industry Verticals repo. The assembly phase (Phase 5) automates all of this.

`{site}` = kebab-case site identifier (e.g., `acme-corp`)
`{SiteName}` = PascalCase (e.g., `AcmeCorp`)
`{Site Display Name}` = human-readable (e.g., `Acme Corp`)
`{target}` = absolute path to the SE's Industry Verticals repo

---

## B. SCS Items — Base Module Updates

Update `{target}/authoring/items/industry-verticals/common/common.module.json`:

### B1. Add site include to `items.includes` array

```json
{
  "name": "sites-{site}",
  "path": "/sitecore/content/industry-verticals/{site}",
  "rules": [
    { "path": "/home", "scope": "SingleItem", "allowedPushOperations": "CreateOnly" },
    { "path": "/Media", "scope": "SingleItem", "allowedPushOperations": "CreateAndUpdate" },
    { "path": "/Data", "scope": "SingleItem", "allowedPushOperations": "CreateOnly" },
    { "path": "/Dictionary", "scope": "SingleItem", "allowedPushOperations": "CreateOnly" },
    { "path": "/Presentation", "scope": "SingleItem", "allowedPushOperations": "CreateOnly" },
    { "path": "/Settings/Site Grouping", "scope": "ItemAndDescendants", "allowedPushOperations": "CreateOnly" },
    { "path": "/Settings", "scope": "ItemAndChildren", "allowedPushOperations": "CreateAndUpdate" },
    { "path": "*", "scope": "Ignored" }
  ]
}
```

### B2. Add media folder rule to `projectMediaFolders` include

```json
{ "path": "/{site}", "scope": "SingleItem", "allowedPushOperations": "CreateOnly" }
```

## C. SCS Items — Site Definition YAML

Create under `{target}/authoring/items/industry-verticals/common/items/sites-{site}/`:

| # | File | Template | Key Fields |
|---|------|----------|-----------|
| 30 | `{site}.yml` | Headless Site (`e5b045ab-...`) | Modules (15 GUIDs from existing site), Name, SiteMediaLibrary, SiteTemplate |
| 31 | `{site}/Home.yml` | Route item | `__Renderings` XML composing all renderings in placeholders |
| 32 | `{site}/Data.yml` | Folder | Container for datasource items |
| 33 | `{site}/Media.yml` | Folder | Container for media references |
| 34 | `{site}/Dictionary.yml` | Folder | Container for dictionary entries |
| 35 | `{site}/Presentation.yml` | Folder | Container for presentation items |
| 36 | `{site}/Settings.yml` | Settings template (`4628b2c8-...`) | Name=`{site}`, FilesystemPath=`/dist/{site}`, AppTemplate, RenderingsPath, PlaceholdersPath |
| 37 | `{site}/Settings/Site Grouping.yml` | Folder | Container for site grouping items |
| 38 | `{site}/Settings/Site Grouping/{site}.yml` | Site Grouping | SiteName=`{site}`, RenderingHost=`{site}`, HostName=`*`, Environment=`*` |

### Critical: Modules field (item #30)

The site root item's `Modules` shared field must contain 15 module GUIDs. **Read these from an existing site in the target repo** — they're environment-specific. All current IV sites use the same set:

```
{5A2F6A6F-3028-4210-AAD3-82C1365C4802}
{9F10010E-49E2-4DCD-A5A1-4DA752ADED2A}
{4342A029-0186-4B0D-8959-FFEF4FD998C2}
... (15 total)
```

## D. SCS Items — Per-Component Templates & Renderings

For **each** cloned component, create under `{target}/authoring/items/industry-verticals/common/items/`:

### D1. Data Template (3 + N files per component)

Under `templatesProject/industry-verticals/Components/Page Content/{TemplateName}/`:

| # | File | What it is |
|---|------|-----------|
| 39 | `{TemplateName}.yml` | Template root — inherits from Content Base + Rendering Params Base |
| 39b | `{TemplateName}/__Standard Values.yml` | **REQUIRED** — self-referential item (Template = parent GUID). Can be empty but must exist. |
| 40 | `{TemplateName}/Data.yml` | Template section — groups fields |
| 41 | `{TemplateName}/Data/{FieldName}.yml` (one per field) | Template field — Type, Sortorder, Source |

### D2. Parameters Template (per component — NOT shared)

Under `templatesProject/industry-verticals/Components/Page Content/{TemplateName}/Rendering Parameters/`:

| # | File | What it is |
|---|------|-----------|
| 41b | `{TemplateName} Parameters.yml` | Rendering parameters template — inherits from SXA rendering params base |
| 41c | `{TemplateName} Parameters/__Standard Values.yml` | **REQUIRED** — Standard Values for the parameters template |

### D3. Datasource Folder Template (per datasource-driven component)

Under `templatesProject/industry-verticals/Components/Page Content/{TemplateName}/`:

| # | File | What it is |
|---|------|-----------|
| 41d | `{TemplateName} Folder.yml` | Folder template — allows editors to create datasource folders in Data |
| 41e | `{TemplateName} Folder/__Standard Values.yml` | **REQUIRED** — Standard Values for folder template |

## F. Per-Site Content Modules

Create under `{target}/authoring/items/industry-verticals/sites/{site}/`:

| # | File | Namespace | References |
|---|------|-----------|-----------|
| 45 | `{site}-content.module.json` | `Project.{SiteName}-Content` | `["Project.IndustryVerticals"]` |
| 46 | `{site}-media.module.json` | `Project.{SiteName}-Media` | `["Project.IndustryVerticals"]` |

Content module includes: `{site}-home`, `{site}-media`, `{site}-data`, `{site}-dictionary`, `{site}-presentation` — all `DescendantsOnly`, `CreateAndUpdate`.

Media module includes: media library path — `DescendantsOnly`, `CreateUpdateAndDelete`.

## G. Config Updates — xmcloud.build.json

Update `{target}/xmcloud.build.json`:

### G1. Add rendering host

```json
"{site}": {
  "path": "./industry-verticals/{site}",
  "nodeVersion": "22.11.0",
  "jssDeploymentSecret": "110F1C44A496B45478640DD36F80C18C9",
  "enabled": true,
  "type": "sxa",
  "buildCommand": "build",
  "runCommand": "next:start"
}
```

### G2. Add modules to postActions

Add to `postActions.actions[0].args.modules`:
```json
"Project.{SiteName}-Content",
"Project.{SiteName}-Media"
```
